# Getting Started

Welcome to **MetalLabs Framework**.

This repository is organized as a laboratory rather than a traditional software project. Each section has a specific purpose and should be explored independently.

There is no required reading order, but if this is your first visit, the following path is recommended.

---

## 📖 1. Framework

Start here.

The Framework contains the principles, workflows and methodologies that define how this laboratory operates.

These documents describe **the process**, not the experiments.

---

## 🧪 2. Chaos Labs

Once you understand the workflow, visit Chaos Labs.

This is where ideas become experiments.

Every experiment follows a documented methodology and attempts to answer a specific question through practical observation.

---

## 🤖 3. Agent Dossiers

Experiments produce observations.

Agent Dossiers accumulate those observations over time.

Rather than ranking AI models, these documents record recurring strengths, weaknesses and behavioural patterns observed while working with different agents.

---

## 🛠️ 4. Tools

The Tools section contains reusable assets that support the workflow.

These may include:

* Agent definitions
* Context templates
* Documentation templates
* Helper scripts
* Experimental utilities

Tools are created only when they solve a recurring problem encountered during real development.

---

## 📝 5. Articles

Finally, Articles contains reflections, essays and lessons learned throughout the journey.

Unlike the Framework, these documents are intentionally opinionated and exploratory.

They document observations rather than define methodology.

---

# A Note About AI Agents

The AI agents presented throughout this repository represent **roles within my personal workflow**, not recommendations for specific models.

The Architect, Auditor, Archivist or Explorer can be performed by any AI model—or even by a human.

The workflow is built around **responsibilities**, not around AI providers.

Replace the agents.

Keep the roles.

---

# A Note About This Project

MetalLabs Framework is not intended to teach "the correct way" to build software with AI.

It documents one developer's ongoing attempt to build a reproducible, maintainable and enjoyable workflow while developing real software projects.

Take what is useful.

Question everything else.

Draw your own conclusions.
