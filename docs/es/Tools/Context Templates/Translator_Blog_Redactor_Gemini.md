El "Traductor" (De Nota Técnica a Post para Humanos)Usa este prompt cuando hayas subido un registro de cambios (changelog), un log de errores o documentación de una API, y quieras explicarle a tu audiencia por qué es importante esa actualización.Prompt:*"Actúa como un redactor técnico de tecnología y divulgador. Revisa [menciona el archivo o la función, ej: el log de desarrollo de la versión 2.0] en nuestro cuaderno. Redacta una entrada de blog corta (unas 400 palabras) explicando este cambio.Sigue esta estructura:El Problema: Qué fallaba o qué limitación teníamos antes (búscalo en la documentación conceptual).La Solución: Qué cambia técnicamente con esta actualización (usa el registro de desarrollo).El Beneficio: Qué gana el usuario final con esto de forma práctica.Mantén un tono cercano, entusiasta y evita la jerga excesiva. Si es necesario usar un término técnico, explícalo con una analogía simple."*
