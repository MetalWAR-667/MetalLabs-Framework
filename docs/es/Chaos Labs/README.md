🧪 Chaos Labs
Chaos Labs es la rama experimental de MetalLabs Framework.

Aquí es donde las ideas dejan de ser suposiciones y empiezan a convertirse en experimentos.

Su propósito no es determinar qué modelo de IA es "el mejor".

Su propósito es observar cómo se comportan diferentes agentes cuando trabajan bajo las mismas restricciones, con la misma información y hacia el mismo objetivo.

Cada experimento está diseñado para responder a una pregunta concreta.

Cada resultado debe ser reproducible.

Cada conclusión debe estar respaldada por evidencias, no por opiniones.

Objetivos
Chaos Labs investiga preguntas como:

¿Cómo abordan diferentes agentes de IA el mismo problema de ingeniería?

¿Qué tipo de tareas se adaptan mejor a cada modelo?

¿Cuánto contexto se necesita antes de que pueda comenzar un trabajo útil?

¿Cómo influyen las diferentes estrategias de prompting en el resultado?

¿Qué nivel de revisión humana es necesario después de cada propuesta?

¿Qué flujos de trabajo siguen siendo efectivos independientemente del proveedor de IA?

El objetivo no es clasificar modelos.

El objetivo es mejorar el proceso de desarrollo.

El Chaos Budget
Todos los experimentos siguen un protocolo común llamado The Chaos Budget.

El protocolo define las restricciones, los límites de mutación, los criterios de evaluación y el proceso de revisión compartidos por cada experimento.

Esto garantiza que los resultados sigan siendo comparables incluso cuando participen diferentes modelos de IA.

El protocolo completo está documentado en:

ChaosBudget.md

Experimentos
Cada experimento se documenta de forma independiente.

Los experimentos típicos incluyen:

observaciones de comportamiento;

evaluaciones de flujos de trabajo;

estudios de ingeniería de prompts;

ingeniería de contexto;

comparaciones de implementación;

análisis de coste de revisión;

observaciones a largo plazo.

Cada experimento recibe su propio documento y se mantiene autocontenido.

Una nota sobre las conclusiones
Chaos Labs no pretende establecer verdades universales.

Cada observación se realiza en un contexto específico, utilizando herramientas, proyectos y restricciones particulares.

Se anima a los lectores a cuestionar la metodología, reproducir los experimentos cuando sea posible y sacar sus propias conclusiones.

El objetivo no es demostrar quién tiene razón.

El objetivo es entender qué se puede aprender.
