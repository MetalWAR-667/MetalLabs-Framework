# 📖 Framework

El Framework contiene los principios, metodologías y documentación a largo plazo que definen cómo opera MetalLabs.

A diferencia de **Chaos Labs**, que se centra en experimentos, el Framework documenta las ideas que han demostrado ser lo suficientemente útiles como para formar parte del laboratorio mismo.

Representa la comprensión actual del proceso de desarrollo.

A medida que nueva evidencia esté disponible a través de la experiencia práctica y la experimentación, el Framework evoluciona en consecuencia.

---

## Proposito

El objetivo del Framework no es prescribir cómo se debe desarrollar el software.

Su propósito es documentar un flujo de trabajo estructurado que ha surgido mientras se construían proyectos de software reales con colaboración asistida por IA.

Cada documento contenido aquí debería responder a una de las siguientes preguntas:

- ¿Cómo opera el flujo de trabajo?
- ¿Qué principios guían el proceso?
- ¿Cómo se toman las decisiones arquitectónicas?
- ¿Cómo se distribuyen las responsabilidades?
- ¿Qué prácticas han demostrado ser valiosas con el tiempo?

---

## Relación con Chaos Labs

Chaos Labs explora ideas.

El Framework las consolida.

Un experimento no pasa automáticamente a formar parte de la metodología.

Solo las observaciones que demuestran repetidamente valor práctico deberían influir en el Framework.

Esta separación permite que la experimentación siga siendo flexible mientras el flujo de trabajo mismo se mantiene estable y predecible.

---

## Documentación viva

El Framework se mantiene intencionadamente como una colección de documentos vivos.

A medida que los flujos de trabajo evolucionan, las herramientas cambian y se aprenden nuevas lecciones, estos documentos pueden ser revisados, simplificados o ampliados.

El objetivo no es preservar la historia.

El objetivo es preservar la comprensión.

---

## Documentos actuales

Los documentos típicos que se encuentran en esta sección incluyen:

- Flujos de trabajo de desarrollo
- Principios de ingeniería
- Metodologías
- Ingeniería de contexto
- Estándares de documentación
- Procesos de toma de decisiones
- Plantillas reutilizables

Aparecerán documentos adicionales de forma natural a medida que el laboratorio crezca.

---

## Principio rector

El Framework debe permanecer independiente de cualquier modelo de IA o plataforma de software en particular.

Los modelos pueden evolucionar.

Las herramientas pueden desaparecer.

El proceso debe seguir siendo comprensible.

---

> *Los experimentos descubren ideas.*
>
> *El Framework preserva las que merecen ser conservadas.*
