## Experimental Project

MetalLab tests will be conducted on **MetalWar Installer AIS**, a customizable Python installer conceived as a deliberately over-the-top homage to the Demoscene of the 1990s.

Repository:

`https://github.com/MetalWAR-667/MetalWar-Installer`

MetalWar Installer is not merely a file-copying utility. The project combines installation logic, audio playback, musical synchronization, real-time graphical effects, user interface, system operations, and executable generation via PyInstaller.

### Why MetalWar Installer?

The choice is based on three main reasons.

**1. It is a disposable project**

MetalWar Installer is not part of the critical architecture of *Lands of Folklore*. It can be modified, broken, restored, or even discarded without compromising the main project.

This allows experimentation with AI agents within a real environment, but without putting important systems at risk.

**2. It is absurdly over-engineered**

Its complexity far exceeds what a conventional installer would need.

This is precisely what makes it useful as a laboratory: it contains enough systems, dependencies, and extravagant decisions to provoke interesting situations during maintenance, refactoring, creativity, and scope control testing.

The exaggeration is not an accidental flaw of the project. It is part of its identity and a deliberately exploitable feature for MetalLab.

**3. It contains clearly differentiated technical domains**

The repository offers several independent intervention surfaces:

| Module              | Role within the chassis                                                                          |
| ------------------- | ------------------------------------------------------------------------------------------------- |
| `main.py`           | The crankshaft. Main loop, synchronization with the musical clock, and state machine.             |
| `config.py`         | The octane. Global parameters, neon colors, and mathematical constants.                           |
| `audio.py`          | The intake. Tracker module loading, channel management, and robotic voice.                        |
| `effects.py`        | The nitrous injection. Starfield, 3D transformations, and CRT effects.                            |
| `ui.py`             | The bodywork. ASCII logo, tactical HUD, and visual system representation.                         |
| `installer.py`      | The pistons. File copying, path detection, and threaded execution.                                |
| `utils.py`          | The toolbox. Resource paths, common utilities, and procedural failure generation.                 |
| `Compiler_GUIV2.py` | The hydraulic press. OneFile executable generation via PyInstaller.                               |

This separation allows designing tests across different capabilities without changing projects:

* logic and state control;
* concurrency and system operations;
* graphics and mathematics;
* audio and synchronization;
* user interface;
* configuration;
* packaging and distribution;
* monolithic code refactoring.

### Role within MetalLab

MetalWar Installer will act as a **common experimental subject** for participating agents.

Each agent will work on an isolated copy of the same initial state, following an equivalent protocol. Modifications will be made in separate branches or environments, preventing one agent from knowing or inheriting solutions generated by others.

The goal is not to indefinitely improve the installer.

Its purpose is to provide a sufficiently real, varied, and disposable environment in which to observe:

* how each agent interprets a task;
* how well it respects scope;
* what degree of mutation it introduces;
* how it protects or alters the existing architecture;
* what quality and review cost it produces;
* and what capacity for invention it shows when constraints are relaxed.

MetalWar Installer is not the final product of MetalLab.

It is the sandbox.
