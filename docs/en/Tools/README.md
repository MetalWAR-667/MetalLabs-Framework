# 🛠️ Tools

The **Tools** section contains reusable resources that support the MetalLabs Framework workflow.

Unlike the **Framework**, which defines principles and methodologies, or **Chaos Labs**, which documents experiments, this section contains practical assets created to solve recurring problems encountered during real software development.

Tools are developed only when experience demonstrates that a repetitive task, workflow bottleneck or documentation process can be improved through reuse.

---

## Purpose

The objective of this section is to reduce repetitive work while preserving human decision-making.

Typical resources may include:

* Agent configurations
* Context templates
* Prompt templates
* Documentation templates
* Workflow utilities
* Helper scripts
* Automation tools
* Experimental applications

Every tool should exist because it solved a real problem—not because the technology made it possible.

---

## Philosophy

Tools are intended to assist the workflow, not define it.

The workflow should remain understandable even if every tool in this directory disappeared tomorrow.

Whenever possible, tools should remain independent of any specific AI provider, model or software platform.

---

## Reusable Assets

Many resources in this directory are designed to be reused across different projects.

Examples include:

* Context templates for common engineering tasks.
* Agent role definitions.
* Prompt libraries.
* Documentation skeletons.
* Project initialization templates.
* Experiment templates.

These assets are expected to evolve continuously as new workflows emerge.

---

## Future Direction

Some ideas currently being explored include:

* Context Builders
* Workflow Assistants
* Documentation Helpers
* AI Agent Configurations
* Prompt Libraries
* Engineering Utilities

Whether these ideas become standalone tools will depend entirely on practical need.
