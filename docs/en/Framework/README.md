# 📖 Framework

The Framework contains the long-term principles, methodologies and documentation that define how MetalLabs operates.

Unlike **Chaos Labs**, which focuses on experiments, the Framework documents the ideas that have proven useful enough to become part of the laboratory itself.

It represents the current understanding of the development process.

As new evidence becomes available through practical experience and experimentation, the Framework evolves accordingly.

---

## Purpose

The objective of the Framework is not to prescribe how software should be developed.

Its purpose is to document a structured workflow that has emerged while building real software projects with AI-assisted collaboration.

Every document contained here should answer one of the following questions:

* How does the workflow operate?
* Which principles guide the process?
* How are architectural decisions made?
* How are responsibilities distributed?
* Which practices have proven valuable over time?

---

## Relationship with Chaos Labs

Chaos Labs explores ideas.

The Framework consolidates them.

An experiment does not automatically become part of the methodology.

Only observations that repeatedly demonstrate practical value should influence the Framework.

This separation allows experimentation to remain flexible while keeping the workflow itself stable and predictable.

---

## Living documentation

The Framework is intentionally maintained as a collection of living documents.

As workflows evolve, tools change and new lessons are learned, these documents may be revised, simplified or expanded.

The objective is not to preserve history.

The objective is to preserve understanding.

---

## Current documents

Typical documents found in this section include:

* Development workflows
* Engineering principles
* Methodologies
* Context engineering
* Documentation standards
* Decision-making processes
* Reusable templates

Additional documents will appear naturally as the laboratory grows.

---

## Guiding principle

The Framework should remain independent of any particular AI model or software platform.

Models may evolve.

Tools may disappear.

The process should remain understandable.

---

> *Experiments discover ideas.*
>
> *The Framework preserves the ones worth keeping.*
